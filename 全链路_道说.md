# 期末复习 全链路 · 道说

> 道生一，一生二，二生三，三生万物。
> 此处之道：以雨课堂之 PDF 为始，复习之全表为终，
> 中经六关：**提文 → 聚合 → 素材 → 汇编 → 道喂 → 全图**。
>
> 又有**闭环之道**：以课夹为本，单课自举——见末附 § 十。
> *反者，道之动也*：从初始 PDF 至最终底层复习资料，无外赖，闭于一夹。

---

## 一 · 设计哲学

道法自然。此链路之四训：

1. **图难于其易，为大于其细**
   不一气呵成万事，分关而立，每关只做一事，
   关与关之间以 JSON / markdown 静态文件相接，断点续跑。

2. **其安也，易持也；其未兆也，易谋也**
   每关皆增量友好。已成者跳之，仅作新增。
   下层数据成静态文件，上层可重跑而不损下层。

3. **不贵其师，不爱其赍**
   底层之器 provider 中立 —— OpenAI 兼容协议为公约，
   DeepSeek、通义、Moonshot、智谱、Ollama 皆可接入。
   不绑实现，留接口，今易明日易换。

4. **道恒无为，而无不为**
   工作流之事尽于此；理解、生成内容之事，留给 LLM / 人。
   此链路所做：把"乱卷之 PDF 山"理为"按章节有序之图集 + 待填占位 + 道喂任务包"。

## 三 · 各关之职责

### ［二］`pdf_提文.py` —— 其安也，易持也

PDF → 图像 + 元数据。直取嵌图（PPT 转 PDF 之 PDF 内含原 jpg/png，PyMuPDF 可直读，比渲染快 10×）。

```bash
python pdf_提文.py                 # 全跑（增量）
python pdf_提文.py --filter 环境   # 仅含"环境"之课
python pdf_提文.py --rescan        # 仅重扫章节信号（秒级）
```

### ［三］`课件_章节聚合.py` —— 其未兆也，易谋也

把同章 PDF 归一，识跨章，标主辅版。多归属之 PDF（`all_chapter_nums = [4, 5]`）归两章。

```bash
python 课件_章节聚合.py
python 课件_章节聚合.py -f 环境毒理
```

### ［四］`课件_知识素材.py` —— 其脆也，易判也

每章生一 markdown 素材文件，含主版图链、辅版概要、复习占位。

```bash
python 课件_知识素材.py
python 课件_知识素材.py --inline-images  # 主版每页内联展示
```

### ［五］`课件_道喂.py` —— 道恒无为，而无不为

**双路径之关**。给一章主版图 + 元信息，让 LLM 填"复习要点"占位。

**离线路径**（不调 API，导出任务包）：
```bash
python 课件_道喂.py --列                        # 列可填章节
python 课件_道喂.py -c 环境法学 --章 2          # 单章离线导出
python 课件_道喂.py -c 环境法学 --章 2 --拼网格 # 同上 + 拼网格图
```
产 `.道喂任务/<课>/<章>.json` + `<章>_grid.jpg`，可手喂任意 LLM playground。

**在线路径**（须 `.env` 配 API key）：
```bash
python 课件_道喂.py --在线 --拼网格              # 全课全章
python 课件_道喂.py -c 环境毒理 --在线 --拼网格 # 一课
python 课件_道喂.py -c 环境毒理 --章 3 --在线   # 一章
python 课件_道喂.py --在线 --dry                # 预估，不真调
```
LLM 返 JSON → 自动回写章 md 之"复习要点"区段（不破坏其他区段）。

**回写策略**：定位"## 复习占位"区段，以"## 复习要点 · LLM 协填"替之；其余原封不动。

### ［五·五］`课件_图谱.py` —— 知常曰明，知明曰归根

**可选关**。据章节复习要点，以 LLM 生成 markmap + mermaid 双格式思维导图，回写至章 md 之"思维导图"区。

**两路径**：

```bash
python 课件_图谱.py --列                            # 列可生成章节 + 复习要点字数
python 课件_图谱.py --课 环境法学 --章 2            # 单章离线导出图谱任务
python 课件_图谱.py --课 环境法学 --在线            # 全章在线生成
python 课件_图谱.py --课 环境法学 --回填            # 从 .道喂任务/<课>/图谱/_resp/ 回填
```

回写策略：定位章 md 中之「## 思维导图占位」或「## 思维导图 · LLM 生成」区段，以完整 markmap + mermaid 内容替之；其余区段原封不动。

**与 课程_闭环.py 集成**：`--图谱 offline|online|writeback` 参数可直接触发此关。

### ［六］`期末_综合汇编.py` —— 执大象，天下往

每课生 5 份汇编：`_综合复习资料.md`（图文+嵌入子图+思维导图·新）/ `_最终复习资料.md`（同，兼容旧名）/ `_期末骨架.md` / `_期末速查.md` / `_LLM_提示语.md`。

`_综合复习资料.md`：① 章级入口表（含子图数/要点完成/图谱完成标志）② 全章综合汇编（要点+嵌入子图折叠+思维导图折叠）。复习要点 0/N 章已填时，各章给出可直接运行之命令提示。

```bash
python 期末_综合汇编.py
python 期末_综合汇编.py -f 流体力学
```

### ［七］`期末_全图.py` —— 圣人执一，以为天下牧

跨课汇为一图，作学期复习总入口。

```bash
python 期末_全图.py                 # 默：解析仓库/_学期总图.md + .json
python 期末_全图.py --json-only     # 仅 json
python 期末_全图.py --md-only       # 仅 md
```

输出 `_学期总图.md` 含：
- 总览（课/PDF/页/章 总数 + 已填进度）
- 各课综览表（教师/学期/章/进度/三件套状态）
- 复习路径建议（4 周计划 + 日均页数）
- 章节进度细化（每课每章 md 大小、已填否）
- **道喂待办**（未填之章按页数排序，附调用命令）
- 速链表

### ［总枢］`期末复习_总枢.py` —— 道生一，一生二

依次跑 取PDF（可选）/提文/聚合/素材/道喂（可选）/汇编/全图。默认不触网、不调 LLM；加参数才动。

```bash
python 期末复习_总枢.py                       # 全链路·增量
python 期末复习_总枢.py --fetch-pdf -f 环境毒理 # 先从雨课堂取 PDF（须登录）
python 期末复习_总枢.py --rescan              # 仅重扫章节
python 期末复习_总枢.py -f 环境毒理           # 仅一课全程
python 期末复习_总枢.py -f 环境毒理 --feed offline --feed-grid # 离线道喂任务
python 期末复习_总枢.py -f 环境毒理 --feed online --feed-grid  # 在线回写要点
python 期末复习_总枢.py -f 环境毒理 --self-boot # 本地 PDF → 最终资料闭环
python 期末复习_总枢.py --only 聚合,素材      # 仅几关
python 期末复习_总枢.py --skip 提文           # 跳某关
```

## 四 · 底层之器

### `_yc_common.py` —— 共用之原子工具

| 函数 | 所司 |
|------|------|
| `log(msg, kind)` / `header(title)` | 彩色控制台输出 |
| `sanitize_filename` | 文件名清洗 |
| `cn_to_int` / `detect_chapter` / `detect_all_chapters` | 章节识别 |
| `parse_pdf_filename` / `parse_course_dirname` | 文件名/目录解析 |
| `read_json` / `write_json` | UTF-8 JSON I/O |
| `section_span_fenced(text, heading_re)` | 定位 Markdown H2 区段（代码块内 ## 免疫·新） |

### `_llm_道.py` —— LLM provider 中立层

零外部依赖（纯标准库 urllib + json + hashlib），以 OpenAI Chat Completions 协议为公约。
凡可通此协议者皆可：DeepSeek、Moonshot、通义、智谱、Together、Groq、Ollama / vLLM / LM Studio。

```python
from _llm_道 import LLMClient

cli = LLMClient.from_env()              # 默认 profile
out = cli.chat("用一句话述老子之教")

# 视觉 + JSON
cli_v = LLMClient.from_env("vision")
result = cli_v.chat_json(
    "据图按 schema 返章节要点",
    images=["page_001.jpg", "page_002.jpg"],
    schema_hint={"概念": "list", "公式": "list"},
)
```

**核心要素**：
- `LLMConfig.from_env(profile)` —— 三 profile（default/vision/long），落空回退
- `LLMCache` —— 内容寻址磁盘缓存（key=sha256(model+payload)），同问不二答
- `chat() / chat_json()` —— 同步聊 / 强制 JSON 返
- 内置：重试、超时、退避、错误识别（4xx 不重试）

**配置**：复制 `.env.example` → `.env`，填三件套。`.env` 与 `.llm_cache/` 皆已入 `.gitignore`。

### `_图_道.py` —— 图像预处理

```python
from _图_道 import 缩图, 拼网格, 分批喂图, 估_批_tokens

缩图("page_001.jpg", max_dim=1568, out="...")
拼网格(pages, cols=3, cell=512, out="grid.jpg")  # 9 图 → 1 图，token 省 92%
for batch in 分批喂图(pages, max_per_batch=20):
    cli.chat(prompt, images=batch.paths)
est = 估_批_tokens(pages, detail="high")  # 粗估 OpenAI Vision token
```

**省 token 之窍**：拼网格——9 张原图 9945 tk，拼一图 765 tk（少 92%）。

### `_道说_提示库.py` —— 程序可用之提示模板

| 任务 | 用例 |
|------|------|
| `章占位填充任务` | 一章主版图 → 结构化「页文转录 + 复习要点」（先文后理） |
| `速查行任务` | 一章 → 一行紧凑（核心/公式/易混/必考） |
| `模拟卷任务` | 一课全章素材 → 完整试卷（选择/填空/名解/简答/论述） |
| `思维导图任务` | 一章复习要点 → markmap + mermaid 双格式思维导图（新） |

每任务含 `system + user_template + schema + parse + render_md + render_placeholder`。一处改之，全链路应。

**`章占位填充任务` 之 schema 含七项**：`页文转录` + `核心概念` + `关键公式` + `重要案例` + `高频考点` + `思考题` + `与他章关联`。LLM 先逐页录原文，后据所录与所见提炼要点。`apply_page_texts(doc_json, 页文转录)` 静态方法把 LLM 录之页文回填 `_doc.json:pages[*].embedded_text`，文层落地——主公可搜可索可摘，agent 下次入提示可直读。详见 § 七 「双方逻辑之资」。

## 五 · 共器之复习实战

### 1. 一日之计：从 PDF 到学期总图

```bash
# 把雨课堂下来的 PDF 放在 雨课堂PDF/<课程>/ 下
python 期末复习_总枢.py
# 18s 后：解析仓库/_学期总图.md 已可读
```

### 2. 三轮复习法（含 LLM 协作）

**第一轮·通览**（约总时之 30%）
- 看 `_学期总图.md`，知本学期总势
- 看 `_期末骨架.md`，知本课结构
- 逐章翻 `_第N章_xxx.md` 之主版图链，对每页之内容有大致印象
- 标疑点（在素材 md 内用 `> [疑]` 注记）

**第二轮·道喂填占位**（约总时之 40%）
```bash
# 配好 .env 后
python 课件_道喂.py --在线 --拼网格
# 或一课一章地喂（更可控）
python 课件_道喂.py -c 环境毒理 --章 3 --在线 --拼网格
```
LLM 自动填章 md 之"复习要点"区。重跑总枢 `--only 全图` 查进度。

**第三轮·速记 + 自测**（约总时之 30%）
- 重跑 `--only 汇编`，使速查表纳入新填之要点
- 主攻 `_期末速查.md` 与 `_LLM_提示语.md`
- 让 LLM 出模拟卷自测

### 3. LLM 协作要点

**调用模式选择**：
| 模式 | 何时用 | 命令 |
|------|--------|------|
| 离线导出 | 暂无 API key、想用 ChatGPT 网页 | `课件_道喂.py --拼网格` |
| 在线·拼网格 | 章 < 50 页、求快 | `--在线 --拼网格` |
| 在线·分页 | 章 > 50 页、求细 | `--在线`（默 max-pages 40） |

**校核**：LLM 易在公式细节、专有名词上漂浮。喂图后看 LLM 填的章 md，亲核重要项。

## 七 · 何不为

此链路**不做**之事（道之"去甚去大去诸"）：

1. **不装独立 OCR 工具**：中文 OCR 工程量大、依赖重、易踩坑。**但让协填之 LLM 顺手录文**——`章占位填充任务` 之 schema 含 `页文转录` 字段，LLM 看图填要点之同时按页录字；回填关同步入 `_doc.json:pages[*].embedded_text`。**一关二事**：token 同费但既得要点又得页文，**无为而无不为**也。
2. **不强生内容**：占位由 LLM 协填，留人主体性。
3. **不做下载器**：雨课堂 PDF 取得是另一关（已有 `雨课堂_PDF_集尽.py`）。
4. **不绑 LLM 实现**：抽象层至上，今 OpenAI 明 DeepSeek 后 Ollama，皆可。
5. **不做 GUI**：命令行 + markdown 即足。
6. **不强求完成度**：缓存 + 增量保证可中断、可重启、可补章。

> 故去彼取此 —— 弃华求实。

### 「双方逻辑之资」之实——文层落于 `_doc.json`

主公之逻辑：可搜可索可摘可编。
agent 之逻辑：可入提示·token 廉·错率低。
**文层**乃二者共用之资。当下落地之路：

```
LLM 协填一章
  ├─ 看主版图（不变）
  ├─ schema 含「页文转录」  ── 让 LLM 逐页录字（不释义、不删冗）
  └─ 返 JSON
       │
       ├─ 章 md 之「## 复习要点」区  ←  render_md 写之（不变）
       └─ _doc.json:pages[*].embedded_text  ←  apply_page_texts 写之（新）
            │
            ├─ 素材 md 重生时·主版图链下展 <details> 折叠之原文（人读）
            ├─ 课夹之 grep 即可搜本课全文（主公可搜）
            └─ agent 下次入提示·直读 embedded_text·无需独靠图（token 大省）
```

**装包 0 · 新关 0 · 改 4 文件**（提示库 + 道喂 + 回填 + 素材）。

## 十 · 闭环之道 —— 反者道之动

> *反者，道之动也；弱者，道之用也。*

前述"七关 + 总枢"，以 `雨课堂PDF/`、`解析仓库/` 为全局之器，多课一仓而治。
然有时则反——**以课夹为本**，单课自成闭环：

```text
<课>/                              # 如 环境法学/、环境毒理学/
├── 00_闭环总览.md                  ← 自动生成之入口（终关之产）
├── 01_原始PDF/  或  02_原始课件PDF/  ← 输入（用户整理之后）
│   └── *.pdf
└── 02_解析成果/  或  03_解析成果/   ← 一至三 + 六 + 七关之产，皆居于此
    ├── _course.json
    ├── _章节图谱.{json,md}
    ├── _素材/
    │   ├── _index.md
    │   ├── _第N章_*.md            ← 含主版图链 + 复习要点（道喂/回填后实）
    │   ├── _期末骨架.md
    │   ├── _期末速查.md
    │   ├── _LLM_提示语.md
    │   └── _最终复习资料.md       ← ★ 终点
    └── <PDF_stem>/page_*.jpg + _doc.json

.道喂任务/<课夹名>/                ← 四 · 道喂之产（全局；与道喂/回填器约定一）
    ├── 第N章_*.json                离线任务包（system / user / schema / 图链）
    ├── 第N章_grid.jpg              拼网格图
    └── _resp/                       ← 五 · 回填之入（手喂 LLM 后置 JSON 于此）
        └── 第N章_*.json             与任务同名之 LLM 响应
```

入口：[`课程_闭环.py`](课程_闭环.py)

```bash
# 列可闭环课夹
python 课程_闭环.py --列

# 单课·增量（自动跳过已成者）
python 课程_闭环.py 环境法学
python 课程_闭环.py 环境毒理学

# 全部可闭环课夹依次
python 课程_闭环.py --全

# 加 LLM 道喂（在线，须 .env 配 API key）
python 课程_闭环.py 环境法学 --feed online --grid

# 加 LLM 道喂（离线，导出 .json+grid.jpg 至全局 .道喂任务/）
python 课程_闭环.py 环境法学 --feed offline --grid

# 喂毕回填（自动扫 .道喂任务/<课>/_resp/）
python 课程_闭环.py 环境法学 --only 回填,汇编,总览

# 仅重扫章节信号（速极快）
python 课程_闭环.py 环境法学 --rescan

# 仅某关 / 跳某关
python 课程_闭环.py 环境法学 --only 素材,汇编,总览
python 课程_闭环.py 环境法学 --skip 道喂,回填

# 单章道喂（避免误填多章）
python 课程_闭环.py 环境毒理学 --feed online --grid --章 3
```

### 闭环之七关

| 关 | 输入 | 产 |
|---|---|---|
| **一·提文** | `01_原始PDF/*.pdf` | `02_解析成果/<stem>/page_*.jpg + _doc.json` + `_course.json` |
| **二·聚合** | `_course.json` | `_章节图谱.{json,md}` |
| **三·素材** | `_章节图谱.json` | `_素材/_第N章_*.md` + `_index.md` |
| **四·道喂**（可选） | 章主版图 + 元 | LLM 在线回写 `## 复习要点` 区  或  `.道喂任务/<课>/*.json + grid.jpg` 离线 |
| **五·回填**（与道喂对偶） | `.道喂任务/<课>/_resp/<章>.json` | 章 md 之 `## 复习要点` 区被实——闭环之实闭 |
| **六·汇编** | 全章素材 | `_素材/_期末{骨架,速查}.md` + `_LLM_提示语.md` + `_最终复习资料.md` |
| **终·总览** | 全态 | `00_闭环总览.md`（重生·七关皆有反映） |

道喂与回填成**对偶**：
- 道喂离线 → 主公自挑闲时喂 LLM → 得 JSON → 置 `.道喂任务/<课>/_resp/<同任务名>.json` → 一令 `--only 回填,汇编,总览` 全成
- 道喂在线（须 `.env`）→ LLM 直接回写章 md → 跳过回填关

### 与原链路之异同

| 视角 | 原链路（七关+总枢） | 闭环之道 |
|---|---|---|
| 起点 | `雨课堂PDF/<课>/*.pdf`（自动下载或归集） | `<课>/01_原始PDF/*.pdf`（已整理之课夹） |
| 终点 | `解析仓库/<课>/_素材/_最终复习资料.md` + `_学期总图.md`（全局） | `<课>/02_解析成果/_素材/_最终复习资料.md` + `<课>/00_闭环总览.md`（局部） |
| 道喂位置 | `.道喂任务/<课>/`（全局） | **同**：`.道喂任务/<课夹名>/`（一脉相承） |
| 范围 | 全工作区跨课总治 | 单课夹独立闭环 |
| 适用 | 雨课堂新下、跨课对照、全学期总览 | 课夹既已整理；专注一课、易迁移、易归档 |
| 相生 | 互补：原链路供全局视野 | 闭环供单课深入 |

### 何以"反"为道之动

- **正链路**：散 PDF 山 → 全局仓 → 各课析 → 总览
- **反链路**：单课夹（既有 PDF）→ 本课析 → 本课总览
- 道喂与回填**首尾衔接**：道喂出门，回填还家；如水之循环
- 一为"由万归一"，一为"由一返万"；二者皆通，皆"无为而无不为"。

### 实战

凡课夹（如 `<课>/`）含 `01_原始PDF/` 或 `02_原始课件PDF/` 中有 PDF，即可入闭环。
增量友好——已成之 `_doc.json` 直跳；既填之"复习要点"区段重生时自动保全。

`00_闭环总览.md` 即一课之入口：含一令贯通命令、闭环状态、章节进度、全局道喂态（任务/回填）、入口速链。
