## 一、空间数据采集与组织概述

### 1.1 空间数据采集

> 将现有的地图、外业观测成果、航空相片、遥感图像、文本资料等转成计算机可以识别处理的数字形式
> 数据采集可分为**属性数据采集**和**图形数据采集**

### 1.2 四种采集方法（简答高频）

| 方法 | 说明 |
|------|------|
| **野外采集** | GPS测量、全站仪等 |
| **地图数字化** | 手扶跟踪数字化、扫描矢量化 |
| **摄影测量** | 航空摄影、数字摄影测量 |
| **遥感处理** | 卫星遥感影像解译与分类 |

### 1.3 地理空间数据 vs 非地理空间数据

| 概念 | 定义 | 关键词 |
|------|------|--------|
| **地理空间数据** | 表示空间实体的**位置、形状、大小**及其分布特征的数据 | 位置+形状+分布 |
| **非地理空间数据（属性数据）** | 表示空间实体的**属性特征**，是对地理空间数据的说明 | 属性+说明 |

### 1.4 地理空间数据三大特征（必考简答）

| 特征 | 说明 | 考试要点 |
|------|------|----------|
| **空间特征** | 空间物体的几何特征以及拓扑关系 | GIS区别于其他系统的根本标志 |
| **属性特征** | 空间对象的属性信息（类型/等级/数量） | 定性+定量 |
| **时间特征** | 地理空间数据是动态的 | 现势性 |

### 1.5 地理空间数据分类

| 类型 | 说明 |
|------|------|
| **图形数据** | 数字化的数据 |
| **图像数据** | 航空和航天数据 |
| **实体属性数据** | 空间实体的专题信息数据 |
| **统计数据** | 描述性数据 |

### 1.6 GIS四种数据组织方式

| 方式 | 核心特征 |
|------|----------|
| **矢量数据** | 点、线、面及拓扑关系 |
| **栅格数据** | 影像图片等规则栅格阵列 |
| **点云数据** | 摄影测量或激光雷达获得 |
| **Geodatabase** | ArcGIS第三代数据模型，面向对象 |

## 二、地理空间数据的基本结构

### 2.1 空间对象抽象

| 维度 | 类型 | 说明 |
|------|------|------|
| **零维** | 点 | 有位置无长度面积 |
| **一维** | 线 | 有长度无面积 |
| **二维** | 面 | 有面积有周长 |
| **三维** | 体 | 有体积 |
| **四维** | 时间 | 时空动态 |

![空间对象抽象](../003-GIS-空间数据-知识点回顾/page_020.jpg)

### 2.2 图形数据的表达

| 类型 | 坐标表达 | 说明 |
|------|----------|------|
| **点** | 单一坐标 (x1, y1) | 空间位置 |
| **线** | 多个坐标序列 | 连接路径 |
| **面** | 多个坐标，首尾坐标重合 | 闭合区域 |

### 2.3 图形数据的拓扑结构（必考名词解释）

> **拓扑关系**：空间实体间的相邻、关联、包含关系。不随地图投影和比例尺变化而改变，是空间分析的基础。

**六类拓扑关系**：点-点、点-线、点-面、线-线、线-面、面-面

**五种基本类型**：邻接、相交、相离、包含、重合

![拓扑关系](../003-GIS-空间数据-知识点回顾/page_026.jpg)

### 2.4 图像（栅格）数据的特征

- 最小单元为**像元/像素**
- 像元由行列号确定位置，具有实体属性的编码值
- **分辨率**：像元大小决定表达精细程度，像元越小越精细，存储空间越大
- 每个栅格单元**只能存在一个值**

![栅格分辨率](../003-GIS-空间数据-知识点回顾/page_031.jpg)

## 三、矢量数据组织

### 3.1 矢量数据结构概述

> 通过记录实体坐标及其关系，尽可能精确地表示点、线、面等地理实体的空间位置和形状
> 坐标空间为**连续空间**，允许任意位置、长度和面积的精确定义

**三大特点**：①数据量少 ②数据精确 ③图形精美

### 3.2 基于文件系统的矢量数据 — Shapefile

> Shapefile是GIS软件界的开放标准，非拓扑简单格式
> **由多个文件组成，前缀必须相同，各文件必须在同一目录下**

| 文件后缀 | 功能 | 必需 |
|----------|------|------|
| **.shp** | 存储要素几何的主文件 | ✅ |
| **.shx** | 存储要素几何索引的索引文件 | ✅ |
| **.dbf** | 存储要素属性信息的dBASE表 | ✅ |
| **.sbn/.sbx** | 存储要素空间索引 | ❌ |
| **.prj** | 存储坐标系信息 | ❌ |
| **.xml** | 元数据 | ❌ |

> ⚡ 考试要点：Shapefile至少由**.shp + .shx + .dbf**三个文件组成

![Shapefile文件组成](../003-GIS-空间数据-知识点回顾/page_037.jpg)

### 3.3 基于地理数据库的矢量数据 — Geodatabase

> ArcGIS第三代数据模型，面向对象，能表示要素的自然行为和要素之间的关系

| 类型 | 存储 | 大小限制 | 适用场景 | 跨平台 |
|------|------|----------|----------|--------|
| **个人Geodatabase** | Microsoft Access (.mdb) | ≤2GB | 个人/小工作组 | 仅Windows |
| **文件Geodatabase** | 文件夹形式 (.gdb) | 单数据集≤1TB（影像可256TB） | 个人/小工作组 | Win+Linux |
| **SDE Geodatabase** | 商业RDBMS（Oracle/SQL Server） | 无限 | 多人/大型企业 | Win+Linux |

**Geodatabase三级结构**：要素类(Feature Class) → 要素数据集(Feature Dataset) → Geodatabase

## 四、栅格数据组织

### 4.1 栅格数据结构概述

> 以规则栅格阵列表示空间对象的数据结构
> 阵列中每个栅格单元的数值表示空间对象的属性特征
> 每个栅格单元**只能存在一个值**

**显著特点**：**属性明显，定位隐含**
- 数据直接记录属性的指针或属性本身
- 所在位置根据行列号转换为相应的坐标给出

### 4.2 栅格数据优缺点

| | 内容 |
|------|------|
| **优点** | 数据结构简单、数学模拟方便、空间叠置组合容易、与遥感数据匹配、易执行空间分析 |
| **缺点** | 数据量大、难以建立拓扑关系、降低分辨率时精度和信息量同时受损、图形质量低 |

### 4.3 栅格数据编码方法（必考简答）

| 编码方法 | 特点 | 适用场景 |
|----------|------|----------|
| **直接栅格编码** | 最简单最直接，逐行逐列记录像元值 | 通用 |
| **游程编码（RLE）** | 记录属性值+连续个数 | 大片同值区域 |
| **四叉树编码** | 递归四等分，同值区合并 | 层次性数据 |
| **链码（Freeman码）** | 用方向码记录边界 | 线状要素 |

> ⚡ 游程编码压缩效果取决于数据的均匀程度；四叉树适合具有区域一致性的数据

## 五、矢量 vs 栅格 对比（必考·简答）

| 对比项 | 矢量数据 | 栅格数据 |
|--------|----------|----------|
| **数据结构** | 严密，冗余小，数据量小 | 简单，易算法实现 |
| **数据量** | 小 | 大 |
| **拓扑关系** | 清晰，易于网络分析 | 难以建立网络连接关系 |
| **叠置分析** | 比较难 | 容易，有利于与遥感匹配 |
| **数学模拟** | 比较困难 | 容易，地理现象模拟 |
| **图形质量** | 精美，精度高 | 粗糙，有锯齿 |
| **存储空间** | 小 | 大 |
| **适宜对象** | 离散对象 | 连续场数据 |

![矢量vs栅格](../003-GIS-空间数据-知识点回顾/page_051.jpg)

> 答题技巧：从数据结构、数据量、拓扑、叠置、模拟、图形、存储、适宜对象8个维度对比

## 六、空间数据库

### 6.1 数据库基础概念

| 概念 | 定义 |
|------|------|
| **数据项** | 定义数据的最小单位，最基本的、不可分的数据单元 |
| **域** | 数据项的取值范围（数值、字符、日期等） |
| **记录** | 由若干相关联的数据项组成，关于一个实体的数据总和 |
| **关键字** | 标识每个记录的数据项 |

### 6.2 四种数据模型

| 模型 | 特点 | 代表 |
|------|------|------|
| **层次模型** | 树状结构，一对多 | 早期GIS |
| **网络模型** | 图状结构，多对多 | 早期GIS |
| **关系模型** | 二维表，最广泛使用 | Shapefile(.dbf) |
| **面向对象模型** | 封装+继承+多态 | Geodatabase |

### 6.3 关系数据库三类完整性

1. **实体完整性**：主键非空原则
2. **参照完整性**：不允许引用不存在的实体
3. **用户自定义完整性**：具体应用所涉及数据必须满足的语义要求

### 6.4 数据库范式

| 范式 | 要求 |
|------|------|
| **1NF** | 每个属性必须是原子、不可分割的数据项 |
| **2NF** | 在1NF基础上，每个非主属性完全函数依赖关键字 |
| **3NF** | 在2NF基础上，消除传递函数依赖 |
| **BCNF** | 每个决定因素都是候选关键字 |

### 6.5 空间数据质量六要素（简答高频）

| 质量要素 | 说明 |
|----------|------|
| **位置精度** | 空间位置的准确程度 |
| **属性精度** | 属性值的准确程度 |
| **逻辑一致性** | 数据间逻辑关系的正确性 |
| **完整性** | 数据是否完整无遗漏 |
| **时间精度** | 数据的时间现势性 |
| **数据世系** | 数据来源和处理历史 |

### 6.6 常见空间索引方法

| 方法 | 说明 |
|------|------|
| **范围索引** | 记录每个空间实体的最大最小坐标 |
| **格网索引** | 将区域划分为大小相等的网格，记录每个网格内空间实体的地址 |
| **四叉树索引** | 递归四等分区域，用莫顿码记录实体位置 |

## 七、ArcGIS操作知识点回顾

### 7.1 MXD文件操作

- **更改数据框/层名称**：单击选定→再次单击进入编辑→输入新名称（不修改Windows文件名）
- **调整数据层顺序原则**（必考）：
  1. 按点、线、面要素类型依次**由上至下**排列
  2. 按要素重要程度高低依次**由上至下**排列
  3. 按要素线划细粗依次**由下至上**排列
  4. 按要素色彩浓淡依次**由下至上**排列

### 7.2 Geoprocessing环境设置

> 环境设置会影响工具和命令的运行
> 可在应用程序级别或运行工具时设置
> 应用程序级别设置与地图文档一起保存

## 八、经典试题精选

### 名词解释

1. **空间数据**：描述空间实体的位置、形状、大小及分布特征的数据，具有空间特征、属性特征和时序特征三大基本特征。
2. **拓扑关系**：空间实体间的相邻、关联、包含关系。不随地图投影和比例尺变化而改变，是空间分析的基础。
3. **游程编码（RLE）**：一种栅格数据压缩方法，记录属性值及其连续出现的个数。适合大片同值区域的压缩。
4. **四叉树编码**：递归四等分的层次性数据结构，同值区合并为一个节点，不同值区继续细分。
5. **Geodatabase**：ESRI提出的面向对象的空间数据存储模型，三级结构：要素类→要素数据集→Geodatabase。
6. **元数据**：描述数据的数据，说明数据的内容、质量、状况和其他有关特征的信息。
7. **DEM**：通过有限地形高程数据实现对地面高程的数字化模拟。表达形式有规则格网（Grid）和不规则三角网（TIN）。
8. **TIN**：由不规则分布的数据点按Delaunay三角准则自动连接形成的三角网络，能根据地形的起伏特征改变采样密度。

### 简答题

**Q1：简述矢量数据与栅格数据的区别（必考）**
> 见第五节对比表。8个维度：数据结构、数据量、拓扑、叠置、模拟、图形、存储、适宜对象。

**Q2：简述栅格数据编码方法**
> ①直接栅格编码（最简单，逐行逐列记录）②游程编码RLE（属性值+连续个数）③四叉树编码（递归四等分，同值合并）④链码/Freeman码（方向码记录边界）

**Q3：简述空间数据的三大基本特征**
> ①空间特征（位置/形状/拓扑关系，GIS区别于其他系统的根本标志）②属性特征（性质/状态/类型/等级/数量）③时序特征（随时间变化的动态特征，现势性）

**Q4：简述空间数据采集的主要方法**
> ①野外采集（GPS/全站仪）②地图数字化（手扶跟踪/扫描矢量化）③摄影测量（航空摄影/数字摄影测量）④遥感处理（卫星遥感影像解译与分类）

**Q5：简述空间数据质量的要素**
> ①位置精度 ②属性精度 ③逻辑一致性 ④完整性 ⑤时间精度 ⑥数据世系
