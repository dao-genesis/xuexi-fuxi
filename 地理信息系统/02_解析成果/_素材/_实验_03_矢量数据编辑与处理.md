# 实验三 · 矢量数据编辑与处理

> 对应理论：第 2 章 空间数据结构 + 第 3 章 空间数据的转换与处理（坐标系变换、拓扑）。本实验为矢量上机的重头戏。

## 一、实验目的
掌握矢量数据的编辑与处理（含**坐标系变换**与**拓扑检查**）的基本流程。

## 二、实验内容（5 个任务，前 5 项均在 ArcCatalog 目录窗口完成）
1. 建立**文件地理数据库**，命名 `Exe04_database`。
2. 新建**要素类（Feature Class）**，坐标系设为 **WGS 1984 World Mercator**。
3. 新建**要素数据集（Feature Dataset）**，并导入外源矢量数据。
4. 导入**栅格（Raster）数据**。
5. 对 `Road.shp` 执行**拓扑分析（Topology）**。

## 三、关键操作（步骤）
1. **建库**：ArcCatalog 右键 `New ▸ File Geodatabase`，重命名为 `Exe04_database`。
2. **建要素类**：库右键 `New ▸ Feature Class`，命名（如 `test_point`），类型选 Point/Polyline/Polygon，`Coordinate System` 选 **Projected ▸ World ▸ WGS 1984 World Mercator**。
3. **建要素数据集并导入**：库右键 `New ▸ Feature Dataset`（同一数据集内所有要素类**必须共用同一坐标系**）；数据集右键 `Import ▸ Feature Class (single/multiple)` 导入外源 shp。
4. **导入栅格**：库右键 `Import ▸ Raster Datasets`。
5. **拓扑分析**（对道路网做完整性检查）：
   - 要素数据集右键 `New ▸ Topology`；设置 **Cluster Tolerance（聚类容差）**；
   - 选择参与的要素类并设 **Rank（等级 1–50，等级越高越不易被移动）**；
   - 添加**拓扑规则**：
     - `Must Not Have Dangles`（线不能有悬挂点——防断头路）
     - `Must Not Have Pseudo Nodes`（不能有伪节点）
     - `Must Not Intersect`（不能相交 / 自相交）
   - `Validate Topology` 验证 → 用 **Error Inspector** 查看并修正错误。

## 四、拓扑错误类型（必考）
- **缝隙（Gap）**：相邻面之间不该有的空隙。
- **叠盖 / 重叠（Overlap）**：要素相互覆盖。
- **悬挂（Dangle）**：线端点未连接到其他线（断头）。
- **伪节点（Pseudo Node）**：本应连续的线被无意义地打断。
- 其它：超出（Overshoot）、未及（Undershoot）、跨越。

## 五、考点与要义
- **要素数据集**内要素类共享同一空间参考，是建立拓扑 / 网络的前提。
- **坐标系变换（投影变换）**：Project 工具把数据从一种坐标系转到另一种；`Define Projection` 仅声明而不变换坐标值——二者勿混淆。
- **Cluster Tolerance** 决定多近的坐标被视为重合；过大会误并要素。
- **Rank** 决定验证时哪条要素被吸附移动。
